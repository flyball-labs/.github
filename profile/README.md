# Flyball Labs

We're an open source project specializing in building future-proof tools for the hardware engineering. Right now the primary focus is on embedded controls and control systems.

## What are control systems?

Controls systems live at the intersection between hardware, software, and the real world. Embedded software reads sensors and drives actuators that affect a system operating in the real world. Examples of this are a plane's fly-by-wire system, a car's automatic braking system, or a quadcopter drone's stability control. These are unique systems not just because they're complicated software and hardware but because the system they're controlling (the plant) have complicated physical dynamics that can be difficult to model or reason about like airflow and aerodynamics or tire and road dynamics.

## What is Flyball Labs doing?

These systems are often developed by deep domain experts: mechanical or aerospace engineers, control theorists, or physicists. Historically the development tools and processes used are complicated, expensive, and proprietary software that are inscrutable to even technical people outside of these industries. This has become a hard bottleneck and painpoint for any interesting hardware controls. There are more small companies and startups that want to work in this space. The hiring pool of control system engineers is artificially constrained by closed source processes. Most significantly, these processes have not innovated due to their position and artificial moat.

Flyball Labs is aiming to create open-source tools for hardware controls engineering to solve these problems. For inspiration we look both to the legacy controls development process as well as more modern software development processes to find new solutions.

## What does future-proof mean?

**Code Forward** - We don't abstract away the code with proprietary DSLs or cloud database models; we build on top of plaintext programming languages to add controls-specific tooling

**Open Source** - Our tooling is all open source and freely available and always will be
